# HTML to PPTX Desktop Converter вЂ” Progress & Decisions

## Project Goal

Build a standalone Windows 11 desktop application that allows consultants to drag-and-drop HTML files (generated by Claude or other AI engines) and convert them to PowerPoint (.pptx) files. The conversion should be as faithful as the `claude-office-skills` repository achieves in the Claude Code context.

This is a **quick win** supporting a broader strategic initiative to get a consultant population embracing AI-assisted workflows and modern deliverable standards.

## Broader Strategic Context

This app is one piece of a larger initiative that includes:
- **Presentation Design Principles** (doc2-design-principles.md) вЂ” medium-agnostic visual communication guidance for AI-assisted content creation
- **Brand Token Layer** (doc3-brand-tokens.md) вЂ” brand-specific visual identity tokens (currently Accenture) with CSS class structure enabling brand reskinning
- **Technical Output Profile / Guardrails** (uc1-guardrails.md) вЂ” defines the HTML/CSS subset that conversion tooling can handle, prescribing the `<section class="slide" data-slide-number="...">` convention

The guardrails document prescribes a constrained HTML subset (no position:absolute, no box-shadow, no rgba, no gradients). Our ported converter is MORE capable than this subset вЂ” it handles absolute positioning, shadows, rgba transparency, and partial borders. This means:
- Compliant HTML (following guardrails) will convert cleanly
- Non-compliant HTML (from other AI engines or legacy sources) degrades gracefully via our fallback mechanisms

## Architecture Decision: Electron (not PyInstaller)

### Why not PyInstaller?
- Core conversion engine is JavaScript/Node.js, not Python
- Wrapping Node.js inside PyInstaller is fragile and unnecessary

### Why Electron?
- Bundles Chromium + Node.js вЂ” replaces Playwright with hidden BrowserWindow
- Native drag-and-drop UI
- electron-builder produces .exe installer for Windows 11
- pptxgenjs runs natively in Node.js

## Architecture & Design Principles

### Module Structure
- `src/main/main.js` вЂ” Electron entry point, orchestrator (no conversion logic)
- `src/main/preload.js` вЂ” Secure IPC bridge (contextBridge, webUtils.getPathForFile)
- `src/main/security.js` вЂ” CSP, navigation guards, file validation
- `src/main/config.js` вЂ” Centralised user-configurable settings
- `src/extraction/extractor.js` вЂ” Hidden BrowserWindow rendering + style extraction (ported from html2pptx-local.cjs)
- `src/generation/generator.js` вЂ” pptxgenjs PPTX generation (ported from html2pptx-local.cjs)
- `src/renderer/index.html` + `renderer.js` вЂ” Drag-and-drop UI

### Security by Default
| Concern | Mitigation |
|---------|------------|
| Node integration in renderer | `nodeIntegration: false` |
| Context isolation | `contextIsolation: true` |
| Content Security Policy | Strict CSP: no external URLs, local file access only |
| File path sanitisation | Validate extension + normalise before loading |
| External URL blocking | Navigation guards on all BrowserWindows |
| Remote images | Blocked in generator вЂ” local files and data URIs only |
| Path traversal | Resolved paths checked against HTML directory |

### Configuration Module (src/main/config.js)
| Setting | Default | Options |
|---------|---------|---------|
| outputStrategy | 'same-directory' | 'same-directory', 'save-dialog', 'fixed-folder' |
| divTextHandling | 'fallback' | 'fallback' (render div text), 'strict' (skip + warn) |
| placeholderRendering | 'visible' | 'visible' (grey shape), 'hidden' (skip) |
| placeholderFillColor | 'D9D9D9' | Any 6-char hex |
| placeholderFillTransparency | 50 | 0вЂ“100 |

Designed for future persistence (JSON file) and settings UI panel.

- [ ] **Gradient detection + rasterisation**: Detect CSS gradients in extracted HTML. ## Key Learnings

1. **Electron v33+ deprecates File.path** вЂ” must use webUtils.getPathForFile() via preload
2. **Don't reinvent tested logic** вЂ” porting html2pptx-local.cjs directly was far more productive than writing a custom generator
3. **The original assumes strict HTML structure** вЂ” text must be in <p>/<h1>-<h6> tags, not bare divs. Our fallback handles this gracefully.
4. **parseInlineFormatting() is the key function** вЂ” builds text run arrays with per-span colour/bold/italic, which pptxgenjs renders as mixed formatting within a single text frame
5. **PptxGenJS margin order** is [left, right, bottom, top] вЂ” NOT CSS order
6. **inset: 0** is critical on text boxes to remove default PowerPoint internal padding
