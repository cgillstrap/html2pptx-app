Common framing deck Bottom-up → ExCo

Lean Portfolio Management: faster, evidence-based prioritisation — without weakening control

This deck is the shared narrative we can use with divisional operations leaders and portfolio leads first, then lift to Divisional CEOs and ultimately Group ExCo. It keeps LPM in business language: value, decisions, outcomes, transparency, and pace.

One-liner: LPM is a decision system that connects strategy → investment → delivery, using lightweight governance and data to optimise value flow.
Sources used to ground this framing: Barclays strategy & tech priorities, Value Realization methodology, and funding-model-to-LPM transition patterns.

1) Why now (and why this lands with Barclays)

Barclays’ declared direction

  • “Simpler, Better and More balanced” as the organising strategy.
  • Technology modernisation to simplify processes, improve agility, scalability and resilience.
  • Responsible AI adoption (incl. broad M365 Copilot rollout; internal AI platform) to improve productivity and service.
Use these as “their words”, not ours, to anchor the narrative.

What this implies for portfolio decisioning

  • More change demand than capacity; greater need to choose.
  • Higher bar on resilience/cyber and risk/control outcomes.
  • Need to translate tech investment into measurable business outcomes and efficiency.
Barclays strategy, tech and AI focus: “Simpler, Better and More balanced”, AI tooling, platform modernisation and resilience emphasis.

2) The problem we’re solving (in business terms)

What leaders experience

  • Prioritisation is slow, political, and re-litigated.
  • Funding is rigid; pivoting spend is expensive and delayed.
  • Too much work in flight; outcomes hard to evidence.

What the system is doing

  • Legacy project funding/approval creates “analysis paralysis”.
  • Value is often defined late and measured at the end (if at all).
  • Operating model shifts (product/value streams) are blocked by old funding mechanics.
Reframe: This is not an “agile adoption” problem. It’s an investment governance and decision cadence problem.
Funding model observations: legacy project-oriented funding is a roadblock; need non-linear, value-indicator-driven funding with guardrails and downstream funding (e.g., value streams).

3) What LPM is (and isn’t)

LPM is

  • A way to align strategy, funding, and execution.
  • A repeatable process for prioritisation trade-offs using transparent criteria.
  • Lightweight governance enabling faster decisions closer to the work.

LPM is not

  • “More SAFe training.”
  • An IT-only methodology change.
  • A loss of financial control (it uses guardrails, not micromanagement).
Audience lens:
COO/Portfolio lead: speed + transparency + fewer escalations CEO: outcomes + competitiveness + capital productivity CIO: links delivery flow to business priorities (less “tech theatre”) ExCo: enterprise alignment + measurable value + resilience

4) The value-led engine (borrowed from Value Realization)

We use the same logic the CFO&EV methodology advocates: define value, hypothesise it, quantify it, and steer it continuously — not once at the start.

Value compass (simple)

  • Define value in the client’s terms (by stakeholder).
  • Value hypotheses + KPIs + targets.
  • Value case that blends quantitative + qualitative benefits.

Value-led governance

  • Set up a lightweight Value Realization Office / value governance pattern.
  • Executive scorecards: snapshot (actual vs target) + trend over time.
  • Use results to re-forecast, pivot, or stop work.
Value Realization journey and VRO governance pattern (continuous value focus; executive reporting; linking KPIs to financial outcomes).

5) The funding & prioritisation shift (crawl → walk → run → fly)

Today

  • Project-centric approvals (e.g., CAR-style gating).
  • Value often loosely defined and measured late.
  • Scope changes require re-approval; pivoting is slow.

Target

  • Capacity-funded product/platform teams within lean budgets + guardrails.
  • Portfolio backlog prioritised against OKRs and constraints.
  • Incremental value measurement and regular steering.

Crawl (near-term)

  • Streamline approvals where possible.
  • Pilot value tracking alongside existing funding.
  • Expand recurring/evergreen funding buckets where appropriate.

Walk / Run / Fly (scale)

  • Pilot LPM funding for selected initiatives; then scale.
  • Move toward capacity-based funding for products/platforms.
  • Embed quarterly/regular portfolio steering tied to OKRs and outcomes.
Transition narrative and crawl/walk/run(/fly) model; guardrails + OKRs + shift to capacity-based product/platform funding.

6) What we measure (minimal set that execs will actually use)

Start simple: a small number of metrics that connect operational KPIs to business outcomes and financial impact, then mature over time.

Value levers (example set)

  • Speed (time-to-market / cycle time)
  • Prioritisation quality (value delivered vs plan; fewer low-value items)
  • Agility (time to pivot funds/resources)
  • Efficiency (reduced abandonment / reduced non-value work)
  • Visibility (clear line of sight cost ↔ value)

Executive reporting pattern

  • Scorecard: snapshot actual vs target
  • Trend: progress over time (glide path)
  • Decision log: pivots/stops + rationale
Cadence is initiative-dependent; the key is a consistent, repeatable rhythm.
Value levers/KPIs and executive reporting guidance drawn from funding model value levers + VR reporting constructs.

7) How the bottom-up path works (your engagement sequence)

Round 1: Divisional COO + Portfolio lead

  • Agree the problem statement and “decision pain”.
  • Select a pilot slice of portfolio to prove value-led steering.
  • Define initial OKRs + guardrails + minimal scorecard.

Round 2: Divisional CEO

  • Translate to strategic outcomes (growth, efficiency, resilience).
  • Confirm sponsorship for governance cadence and decision rights.
  • Align with divisional CIO on enablers (without turning it technical).

Round 3: Group ExCo

  • Show cross-divisional comparability and enterprise view.
  • Propose Group-level guardrails, common metrics, and scaling plan.
  • Secure enterprise sponsorship and investment in the enabling layer.

Parallel: CIO alignment

  • Recast “SAFe/LPM mechanics” into the business decision model.
  • Confirm what changes in governance, funding, and data.
  • Agree how tech modernisation/AI investments map to outcomes.
