Lean Portfolio Management: faster, evidence‑based prioritisation — without weakening control

LPM is a decision system that connects strategy → investment → delivery, using lightweight governance and data to optimise value flow.

One message: Make fewer, clearer investment decisions — faster — with traceable value and stronger outcomes.

This deck is designed to be reused across audiences, with emphasis adjusted per stakeholder group (COO/Portfolio lead, CEO, CIO, Group ExCo).

Why now (and why this should land)

Barclays’ declared direction

  • “Simpler, Better and More balanced” as the organising strategy.
  • Technology modernisation to simplify processes and improve agility and resilience.
  • Responsible AI adoption (incl. M365 Copilot) to improve productivity and service.

Use these as the anchor phrases to avoid “methodology debate.”

What that implies for the change portfolio

  • More good ideas than capacity: prioritisation must get sharper.
  • Resilience and cyber pressures: outcomes must be measurable and governed.
  • Efficiency and customer experience: value must be visible, not assumed.

Outcome: Align decisioning to strategy, improve speed safely, and evidence value continuously.

The problem we’re solving (in business terms)

What leaders experience

  • Prioritisation is slow and re‑litigated; decisions escalate.
  • Funding is rigid; pivoting spend is expensive and delayed.
  • Too much work in flight; outcomes are hard to evidence.

What the system is doing

  • Legacy project approvals create “analysis paralysis.”
  • Value is defined late and measured at the end (if at all).
  • Product/value‑stream shifts are blocked by old funding mechanics.
Reframe: This is not an “agile adoption” problem. It’s an investment governance and decision cadence problem.

What LPM is (and isn’t)

LPM is

  • A way to align strategy, funding, and execution.
  • A repeatable method for transparent trade‑offs.
  • Lightweight governance that enables faster decisions closer to the work.

LPM is not

  • More framework training.
  • An IT‑only change.
  • A loss of financial control (it uses guardrails, not micromanagement).

Audience lens: COOs care about speed and throughput; CEOs care about outcomes; CIOs care about flow and feasibility; ExCo cares about enterprise alignment and resilience.

The value-led engine (the bridge to business stakeholders)

We reframe LPM using a value realisation logic: define value, hypothesise it, measure it, and steer continuously.

Value compass (simple)

  • Define value in the client’s terms (by stakeholder).
  • Value hypotheses + KPIs + targets.
  • A value case combining quantitative + qualitative outcomes.

Value-led governance

  • Lightweight value governance (e.g., VRO pattern) for accountability.
  • Executive scorecards: snapshot (actual vs target) + trend over time.
  • Use results to re-forecast, pivot, or stop work.

The funding & prioritisation shift (crawl → walk → run)

Today

  • Project-centric approvals and gated funding.
  • Scope changes require re‑approval; pivoting is slow.
  • Value often loosely defined and measured late.

Target

  • Capacity-funded product/platform teams within lean budgets + guardrails.
  • Portfolio backlog prioritised against OKRs and constraints.
  • Incremental value measurement and regular steering.

Crawl: introduce value tracking + streamline approvals where possible. Walk/Run: pilot LPM funding, then scale capacity funding and common scorecards.

What we measure (minimal set executives will use)

Start with a small set of metrics that connect operational KPIs to business outcomes, then mature over time.

Value levers (example set)

  • Speed: time-to-market / cycle time.
  • Prioritisation quality: value delivered vs plan.
  • Agility: time to pivot funds/resources.
  • Efficiency: reduced abandonment / reduced non-value work.
  • Visibility: clear line of sight cost ↔ value.

Executive reporting pattern

  • Scorecard: snapshot actual vs target.
  • Trend: progress over time (glide path).
  • Decision log: pivots/stops + rationale.

Cadence depends on initiative type; the point is a consistent rhythm.

How the bottom-up path works (your engagement sequence)

Round 1: Divisional COO + Portfolio lead

  • Agree the problem statement and decision pain.
  • Select a pilot slice of portfolio to prove value-led steering.
  • Define initial OKRs + guardrails + minimal scorecard.

Round 2: Divisional CEO

  • Translate to strategic outcomes (growth, efficiency, resilience).
  • Confirm sponsorship for cadence and decision rights.
  • Align with CIO on enablers (without turning it technical).

Round 3: Group ExCo

  • Show cross-divisional comparability and enterprise view.
  • Propose Group guardrails, common metrics, scaling plan.
  • Secure enterprise sponsorship and enabling investment.

Parallel: CIO alignment

  • Recast “LPM mechanics” into business decision model.
  • Confirm changes to governance, funding, and data.
  • Agree how tech modernisation maps to outcomes.

The ask (what we need from each audience)

COO / Portfolio lead

  • Decide: pilot scope, portfolio boundary, cadence.
  • Provide: top demand themes, constraints, current measures.
  • Nominate: business owners for OKRs and value hypotheses.

CEO / ExCo

  • Decide: decision rights, guardrails, enterprise KPIs.
  • Align: strategic priorities and trade-off principles.
  • Sponsor: scaling across divisions with visible accountability.
